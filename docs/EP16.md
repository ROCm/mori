# EP通信库性能测评 SOP — EP16

# 背景

1.  通信库链接: [mori](https://github.com/amd/mori) (ep-tuning 分支)

2.  测试基于的相关硬件/网卡说明
    - GPU: 8× AMD Instinct MI355X (gfx950, 256 CU) / 节点，共 2 节点
    - 机内互联: XGMI (AMD Infinity Fabric)
    - 机间互联: AMD Pensando AINIC 400Gbps RoCE 网卡
    - FP8 格式: fp8_e4m3 (OCP 标准)

3.  测试环境与复现步骤
    - 环境搭建: 双机各启动 docker 容器，编译安装 mori（`pip install --no-build-isolation -e .`）
    - 在 login node 执行以下命令：
      ```bash
      # quick tuning（默认模式，搜索空间较小但覆盖主要配置）
      bash tools/bench_ep_internode_performance.sh \
          --node0 <node0-fqdn> --node1 <node1-fqdn> \
          --container <container-name>

      # full tuning（搜索空间更大，耗时更长，用于验证 quick 是否遗漏最优点）
      bash tools/bench_ep_internode_performance.sh \
          --node0 <node0-fqdn> --node1 <node1-fqdn> \
          --container <container-name> --tuning-scope full
      ```
    - 脚本自动处理：kernel 选择（v1_ll / v1）、shmem 大小、GPU 空闲检测、结果解析
    - 输出 `bench_results/ep16_<timestamp>/`：`summary.txt`（汇总）+ `raw/`（原始数据）+ `bench.log`
    - 更多选项见 `bash tools/bench_ep_internode_performance.sh --help`

# 测试目标

1.  是否达到官方给出性能

2.  是否达到硬件理论带宽：机内XGMI / 机间RDMA

**硬件理论带宽：**

- **XGMI**：MI355X 每 GPU 有 7 条 XGMI link，每 link 153.6 GB/s（双向）= 76.8 GB/s（单向），单 GPU 总单向带宽 = 7 × 76.8 = 537.6 GB/s
- **RDMA**：AINIC 400Gbps RoCE 网卡，理论单向带宽 = 400/8 = 50 GB/s/卡，8 卡/节点共 8 × 50 = 400 GB/s/节点

3.  与N卡的DeepEP性能对比

# 测试方案

- 使用 `--cmd tuning --tuning-scope quick` 搜索最佳 block_num / warp_per_block / rdma_block_num 配置
- 搜索空间: block_num ∈ {32, 64, 128, 256}, warp_per_block ∈ {4, 8, 16}, rdma_block_num ∈ {bn/2, bn×2/3}
- 每个配置跑 5 轮（跳过 round 0），选择最优配置
- Kernel 选择: < 4096 tokens 用 `v1_ll`（低延迟），≥ 4096 tokens 用 `v1`（高吞吐）
- Dispatch FP8: `--dtype fp8_e4m3`; Dispatch BF16 + Combine BF16: `--dtype bf16`
- Config 格式: `B/W/R` 表示 block_num=B, warp_per_block=W, rdma_block_num=R
- 测试参数: EP16, num_qp=2, hidden_dim=7168, num_experts=256, num_experts_per_token=8

**带宽 / 延迟指标说明：**

| 指标 | 含义 |
| --- | --- |
| RDMA Bandwidth (GB/s) | 仅统计跨节点 RDMA 传输部分的带宽：`rdma_recv_tokens × hidden_dim × dtype_size / time` |
| XGMI Bandwidth (GB/s) | 统计所有 token 传输的带宽（含节点内 XGMI + 节点间 RDMA）：`total_recv_tokens × hidden_dim × dtype_size / time` |
| LL Bandwidth (GB/s) | XGMI BW × LL 修正系数（`max_tokens × num_experts_per_token / total_recv_tokens`） |
| Latency (us) | dispatch/combine 端到端耗时（微秒） |

每个指标报告 Best / Worst / Average（跨所有 round × 所有 rank）。
