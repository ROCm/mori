# EP通信库性能测评 SOP — EP16 (双机 16 卡)

# 背景

1.  通信库链接

2.  测试基于的相关硬件/网卡说明

3.  测试环境+跑通的文档，最好补充测试脚本+数据处理脚本

# 测试目标

1.  是否达到官方给出性能

2.  是否达到硬件理论带宽：机内XGMI / 机间RDMA

3.  与N卡的DeepEP性能对比

# 测试方案

## Kernel 选择

- **小数据量（< 4096 tokens）**：使用 `v1_ll`（InterNodeV1LL），针对低延迟优化的 low-latency kernel
- **大数据量（≥ 4096 tokens）**：使用 `v1`（InterNodeV1），针对高吞吐优化的带宽型 kernel

## 带宽 / 延迟指标说明

测试脚本输出三种带宽指标和延迟：

| 指标 | 含义 |
| --- | --- |
| RDMA Bandwidth (GB/s) | 仅统计跨节点 RDMA 传输部分的带宽：`rdma_recv_tokens × hidden_dim × dtype_size / time` |
| XGMI Bandwidth (GB/s) | 统计所有 token 传输的带宽（含节点内 XGMI + 节点间 RDMA）：`total_recv_tokens × hidden_dim × dtype_size / time` |
| LL Bandwidth (GB/s) | XGMI BW 乘以 LL 模式修正系数（`max_tokens × num_experts_per_token / total_recv_tokens`），反映 LL 场景下的等效带宽 |
| Latency (us) | dispatch/combine 操作的端到端耗时（微秒），取所有 rank 的 CUDA event 计时 |

每个指标报告 Best / Worst / Average：
- **Best**：所有 round × 所有 rank 中的最优值（带宽取 max，延迟取 min）
- **Worst**：所有 round × 所有 rank 中的最差值（带宽取 min，延迟取 max）
- **Average**：所有 round 的 rank 均值再求均值

## 测试参数

- EP size: 16（双机，每机 8 GPU）
- num_qp: 2
- hidden_dim: 7168
- num_experts: 256（每 rank 16 个 expert）
- num_experts_per_token: 8
- bench repeat: 10 rounds（跳过 round 0）
- FP8 dispatch 使用 `fp8_e4m3`（OCP 格式），combine 使用 `bf16`
- Kernel config: block_num=96, rdma_block_num=64, warp_per_block=8（bench 模式使用默认值，未做 tuning）

## 测试脚本

```bash
bash tools/bench_ep_internode_performance.sh \
    --node0 <hostname0> --node1 <hostname1>

# 自定义选项
bash tools/bench_ep_internode_performance.sh \
    --node0 <host0> --node1 <host1> \
    --tokens "128,4096"   \
    --dtypes "bf16"       \
    --output-dir /path/to/output
```
